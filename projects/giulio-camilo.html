Giulio Camilo: A Flashcard Engine Named After a Memory Theater

Matthew Lindley Personal project, 2026

Python Flask Spaced Repetition
Abstract A small, self-hosted flashcard app for spaced repetition, named after Giulio Camillo, the 16th-century philosopher who designed a literal wooden "Memory Theater" meant to hold all human knowledge in spatial form. This version is considerably less ambitious — a Flask app and a JSON file — but the goal is the same: turn scattered facts into something you actually retain.

Motivation

Most flashcard tools are built for someone else's curriculum. I wanted decks built around exactly what I was trying to retain in a given week — consulting frameworks, vocabulary, the basics of the stack I was building this in — without fighting an app's opinions about how a card should look.

Approach

The whole thing is deliberately small:

  • A single data.json defines decks (named groupings) and memory queues — each one a question paired with an optional answer, image, and sound clip.
  • A Flask route reads that file, resolves each card's image and audio paths, and renders the deck.
  • A timer view paces review sessions, since spaced repetition is as much about cadence as content.

There's no database and no auth — the data model is a JSON file because a JSON file is the right amount of structure for a tool only one person uses.

Status

Runs locally, used for my own studying. It's the kind of project that doesn't need a public deployment to be worth having built.

Reflection

The constraint that mattered was resisting the urge to generalize it. It would be easy to add user accounts, a real database, a deck marketplace — none of that was the actual problem. The actual problem was "I keep forgetting this," and a JSON file and a Flask route solved it.

  1. Yates, F. The Art of Memory. University of Chicago Press, 1966 — on Giulio Camillo's Theatro della Memoria and the broader tradition of memory palaces.
  2. Flask — Python web framework, flask.palletsprojects.com
